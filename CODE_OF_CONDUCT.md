# Code of Conduct

## Our Standard

Contributors are expected to keep discussion technical, respectful, and useful. Critique code and ideas directly; do not make it personal.

Examples of expected behavior:

- being precise and constructive in reviews
- explaining tradeoffs clearly
- respecting different experience levels
- focusing on reproducible facts when reporting bugs

Examples of unacceptable behavior:

- harassment or personal attacks
- inflammatory or insulting language
- publishing private information without consent
- derailing technical discussions with hostility

## Enforcement

Project maintainers may remove, edit, or reject contributions or discussions that violate this policy.

## Scope

This applies to repository discussions, issues, pull requests, and other project spaces.
